05

Overfitting Verdict — Is the Backtest Lying?

Overfit

PSO All-Max Boundary

All 8 signal weights converge to exactly 3.0 across every seed. PSO is not learning — it's maxing out. The fitness function rewards all signals equally so the swarm just hits the ceiling.

Overfit

Daily Bars vs Weekly Options

Backtest uses daily OHLC. Real weekly options are traded intraday. A daily bar that shows "Nifty up 50 pts" hides a 200-pt whipsaw that wiped the premium at 10:30 AM.

Overfit

Trend Gate Look-Ahead Bias

close > sma50 AND sma20 > sma50 was TRUE for most of 2020–2023 (bull run). Backtest "works perfectly" because the gate accidentally filters all losing years. In 2026 bearish regime, it blocks everything.

Risk

3-Fold Walk-Forward Weak

3 sequential chunks for walk-forward is statistically underpowered. Need at minimum 5 blocked periods with holdout, or an expanding-window walk-forward (train on years 1–3, test year 4, retrain 1–4, test year 5).

Risk

No Regularization on Weights

PSO weight bounds lo=0.0, hi=3.0 with no L1/L2 penalty. Nothing stops the optimizer from finding weights that fit the training period and fail live. Add penalty = 0.1 * sum(abs(w-1) for w in weights).

Warning

t_exp=3/365 Backtesting Error

Every trade in the backtest uses 3 days to expiry regardless of actual calendar date. This systematically overstates option premiums on non-expiry days and understates theta decay on expiry days.

trail.py — Fix Peak Tracking

class TrailManager: def __init__(self, expiry_mode=False): self.peak = None # running peak, updated each candle self.expiry_mode = expiry_mode self.partial_trigger = 0.20 if expiry_mode else 0.40 self.trail_pct = 0.15 if expiry_mode else 0.20 self.booked = False def update(self, current, entry, qty, initial_qty): self.peak = max(self.peak or entry, current) gain = (current - entry) / entry if gain >= self.partial_trigger and not self.booked: self.booked = True return "PARTIAL_EXIT", qty // 2 trail_sl = self.peak * (1 - self.trail_pct) if self.booked and current < trail_sl: return "FULL_EXIT", qty if gain <= -0.55: return "STOP_LOSS", qty return "HOLD", 0

PSO fitness — add regularization

def fitness_for_pso(df, names, folds=5): n = len(df) # Blocked walk-forward: train on 60%, validate on 20%, test 20% splits = [] chunk = n // folds for k in range(folds - 1): train = df.iloc[:chunk * (k + 1)] val = df.iloc[chunk*(k+1):chunk*(k+2)] splits.append((train, val)) def fitness(vec): weights = dict(zip(names, vec)) # L1 regularization: pull weights toward 1.0 reg = 0.04 * sum(abs(w - 1.0) for w in vec) exps, dds = [], [] for train, val in splits: r = run_backtest(val, weights=weights) if r.get("trades", 0) < 5: return -1e6 exps.append(r["expectancy"]) dds.append(r["max_drawdown_pct"]) mu = sum(exps) / len(exps) sd = (sum((e-mu)**2 for e in exps)/len(exps))**0.5 return mu - sd - 0.3*(sum(dds)/len(dds)) - reg return fitness
